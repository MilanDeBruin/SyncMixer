# SyncMixer

**Sync your own Spotify playlists into one curated playlist — automatically.**

---

## Quick start

1. **Configure your Spotify Client ID**
   Add your own Client ID to `appsettings.json`:

   ```json
   {
     "Spotify": {
       "ClientId": "<YOUR_SPOTIFY_CLIENT_ID>"
     }
   }
   ```

2. **Mark the playlists you want to include**
   Only playlists **you own** are considered.
   To include a playlist in the sync, **end its name with `-s`**.
   Examples: `Roadtrip -s`, `Gym Mix -s`, `Deep Focus -s`.

3. **Run the app**
   Build and run the project. The app will discover your `-s` playlists, fetch tracks, de-duplicate, and update the SyncMixer target playlist.

---

## What the app does

* **Discovers source playlists** you own whose names **end with `-s`**.
* **Creates/updates** a target playlist (e.g., `SyncMixer -m`) and fills it with the combined, de-duplicated tracks from your `-s` playlists.
* **Writes an attribution description** on the created/managed playlist:

  ```
  This playlist is generated by the SyncMixer app.
  Source: Mixer integration.
  ©{CURRENT_YEAR} - Zuipskuur Invent
  ```

  (The year is inserted dynamically at runtime.)

---

## Naming convention (important)

* To **include** a playlist in the merge: end the name with **`-s`** (exact suffix).
* Only **your own** playlists are processed; collaborative or followed playlists you don’t own are ignored.

---

## Configuration

`appsettings.json` expects at least your Spotify Client ID:

```json
{
  "Spotify": {
    "ClientId": "<YOUR_SPOTIFY_CLIENT_ID>"
  }
}
```

> Tip: If your setup also uses Client Secret and Redirect URI, add them under the same `Spotify` section to match your environment.

---

## Notes & recommendations

* If the app creates or modifies playlists on your behalf, make sure your OAuth flow requests the appropriate Spotify scopes (e.g., for private/collaborative playlists).
* The app de-duplicates tracks by Spotify URI, so the final playlist won’t contain duplicates from different sources.
